## 二、飞机系统

### 2.1 ADS-B 广播式自动相关监视

ADS-B（Automatic Dependent Surveillance Broadcast）是一种基于卫星导航和空地数据链的监视技术。飞机自动广播位置、高度、速度、识别码等信息，地面站和周围飞机均可接收。

**ADS-B Out（发射）与 ADS-B In（接收）：**
- Out：飞机每秒广播一次位置信息（GPS经纬度、气压高度+GNSS几何高度、地速、航向、ICAO 24位地址码）
- In：驾驶舱显示周围 ADS-B 目标（CDTI），与 TCAS 互补

**两种数据链：**
| 类型 | 频率 | 适用地区 |
|------|------|----------|
| 1090ES | 1090 MHz | 全球标准（ICAO指定），与Mode S应答机兼容 |
| UAT | 978 MHz | 美国通用航空，支持FIS-B气象服务 |

**与 SSR 对比：**
| 特性 | SSR | ADS-B |
|------|-----|-------|
| 询问方式 | 地面主动询问 | 飞机自主广播 |
| 定位精度 | 0.1-1 NM | 10-30 m |
| 更新率 | 4-12 秒 | 约 1 秒 |
| 覆盖 | 雷达视距 | 可覆盖雷达盲区 |

**运行要点：** ADS-B 自动运行，无需飞行员额外操作。应答机关闭时 ADS-B 也关闭。GNSS失效时位置数据降级。

### 2.2 CPDLC 数据链通信

CPDLC（Controller Pilot Data Link Communications）是通过数据链在管制员和飞行员之间传输结构化文本消息的通信方式，用于执行管制指令。

**底层数据链：**
| 数据链 | 覆盖范围 | 使用场景 |
|--------|----------|----------|
| VDL Mode 2 | 约 200 NM | 大陆航路和终端区 |
| HFDL | 数千公里 | 海洋、极地区域 |
| SATCOM | 全球 | 海洋区域主用 |

**两套标准：** FANS-1/A（波音，成熟全球部署）与 ATN B1（空客/欧洲，ICAO官方标准）。新飞机（B787/A350）同时支持两套。

**消息类型：**
| 方向 | 示例 |
|------|------|
| 管制→机组（Uplink） | CLIMB TO FL310, PROCEED DIRECT PANKI |
| 机组→管制（Downlink） | WILCO, UNABLE, REQUEST CLIMB TO FL350 |
| 系统消息 | Connection Establishment, End Service |

**DCL（数字放行）：** CPDLC在机场最成熟的应用，推出前通过数据链接收放行许可（航路、起始高度、SID、应答机编码），替代语音放行。

**运行要点：** CPDLC不用于紧急情况（仍用MAYDAY语音）、不用于起飞/着陆指令、不用于非标准/非常规指令。洋区运行时CPDLC+ADS-C是标准配置。

### 2.3 PBN 导航体系

PBN（Performance Based Navigation）将关注点从"用什么导航源"转移到"飞这条航路需要多高的导航精度"。分为 RNAV 和 RNP 两类。

**RNAV（区域导航）：** 飞机沿航路点间路径飞行，不依赖飞越地面导航台。无需机载性能监视和告警能力（OPMA）。

**RNP（所需导航性能）：** 在RNAV基础上增加机载性能监视与告警，系统实时计算定位精度，精度不足时自动告警。

| 规范 | 精度（95%） | 场景 |
|------|-------------|------|
| RNAV 10 / RNP 4 | ±10 NM / ±4 NM | 洋区 |
| RNAV 5 | ±5 NM | 大陆航路 |
| RNAV 1 / RNP 1 | ±1 NM | SID / STAR |
| RNP APCH | ±0.3 NM（最终航段） | 进近 |
| RNP AR APCH | 低至 ±0.1 NM | 特殊授权进近 |

**RNP APCH 四种最低标准：**
| 类型 | 垂直引导 | 最低标准 |
|------|----------|----------|
| LNAV | 无 | 最高（400-600ft） |
| LNAV/VNAV | Baro-VNAV | 中等（250-350ft） |
| LP | 无（水平精度更高） | 中等 |
| LPV | SBAS（类精密） | 最低（低至200ft DA） |

**常见误区：** RNAV 不等于 GPS。导航源可以是 GNSS、IRS、DME/DME、VOR/DME。

---

### 2.4 RVSM 缩小垂直间隔

RVSM（Reduced Vertical Separation Minima）将 FL290-FL410 的垂直间隔从 2000ft 压缩到 1000ft，可用高度层从 7 个增加到 14 个。

**核心要求：** 两部独立高度测量系统、高度警戒系统、自动高度控制系统。总高度误差（TVE）不超过 ±200ft，统计偏差（ASE/AAD）各不超过 ±80ft（ICAO Doc 9574）。

**与非 RVSM 飞机间隔：** 非 RVSM 飞机穿越时需保持 2000ft 垂直间隔。

---

### 2.5 TCAS 空中防撞系统

TCAS（Traffic Collision Avoidance System）是机载最后一道安全防线，通过应答机问答独立判断冲突。

**TCAS II Version 7.1（商业航空标准）：**
- TA（交通咨询）：τ ≈ 20-48 秒，黄色实心圆，提示观察
- RA（决断咨询）：τ ≈ 15-35 秒，红色实心方块+语音指令，必须立即执行

**RA 类型：**
| 类型 | 语音 | 要求 |
|------|------|------|
| 预防性 | MONITOR VERTICAL SPEED | 不改变当前垂直速度 |
| 修正性 | CLIMB / DESCEND | 按指令改变垂直速度 |
| 变更性 | CLIMB NOW / DESCEND NOW | 7.1 引入，可反转方向 |

**RA 处置原则：**
1. 立即响应，不分析不犹豫，RA 优先级高于管制指令
2. 柔和但果断的俯仰变化（约 0.25G），垂直速度率 1500-2000 ft/min
3. 不使用航向改变回避
4. 机动后报告管制员"TCAS RA"
5. 冲突解除后语音"CLEAR OF CONFLICT"再返回原高度

**低高度抑制：** 无线电高度 1100ft 以下 RA 自动抑制为 TA only；1100-2350ft 下降指令被削弱。

---

### 2.6 GPWS / EGPWS 近地警告系统

GPWS（Ground Proximity Warning System）用于防范 CFIT（可控飞行撞地）。核心传感器是无线电高度表。

**GPWS 七个模式：**
| 模式 | 名称 | 触发条件 | 语音 |
|------|------|----------|------|
| 1 | 过大下降率 | 近地下降率超标 | SINK RATE → PULL UP |
| 2 | 过大地形接近率 | 地形迅速接近 | TERRAIN → PULL UP |
| 3 | 起飞/复飞后掉高 | 起飞或复飞后丢失高度 | DON'T SINK |
| 4 | 不安全构型 | 高/低高度构型保护 | TOO LOW FLAPS / GEAR |
| 5 | 低于下滑道 | ILS进近显著低于下滑道 | GLIDESLOPE |
| 6 | 决断高度 | 经过DA/DH | MINIMUMS |
| 7 | 风切变警告 | 探测到风切变 | WINDSHEAR |

**EGPWS 增强功能：**
- 全球地形数据库：比对飞机位置与数字地形图，"看得见前方的山"
- FLTA（前视地形回避）：预测未来轨迹，提前 30-60 秒告警
- PDA（过早下降警告）：进近中高度不足以飞到跑道时触发
- SmartRunway：自动识别跑道位置，提示跑道过长/障碍物
- SmartLanding：检查跑道长度是否足够

**PULL UP 记忆项目：**
1. 断开自动驾驶和自动油门
2. 最大推力（TOGA）
3. 按正常姿态抬头（15-20°），不抖杆不失速
4. 验证推力、确认正上升率
5. 按 FCOM 收襟翼
6. 结合地形显示避让

**与 TCAS 冲突时：** GPWS PULL UP 凌驾于 TCAS RA 之上。

## 三、航空气象

### 3.1 METAR 读取顺序（按决策优先级）

1. **风、能见度/RVR、天气现象** — 决定能否起降、跑道、进近方式
2. **云（云底高、云量）** — 决定进近方式选择
3. **温度/露点、QNH** — 温露差 ≤ 3°C 且温度 ≤ 0°C 关注结冰
4. **最近一小时变化（RE组/WS组）** — 趋势判断

**实操提示：** 同时读当前报和上一份例行报，对比趋势。

---

### 3.2 侧风与阵风评估

**侧风心算：** 夹角 30°→风速×50%；45°→×70%；60°→×90%。

**低空风切变信号：** 机场上游 20nm 内雷暴回波+地面风方向变化≥30°；强逆温层；跑道两端风速差>10kt。

**阵风注意：** 阵风>15kt 且与跑道夹角>30°，侧风限制需降级使用（湿跑道×0.8）。

---

### 3.3 能见度与 RVR

**核心原则：** 进近最低标准绑定的是 RVR，不是气象能见度。

| 进近类型 | RVR 最低标准 |
|----------|-------------|
| CAT I | 550m |
| CAT II | 300-350m |
| CAT IIIA | 200m |
| CAT IIIB | 50-200m |
| 非精密/类精密 | 800-1200m |

**着陆标准：** 以跑道入口点和中间点RVR中较低值为准。

---

### 3.4 TAF 解读策略

**时间限定词：**
| 词 | 含义 |
|----|------|
| FM | 从某时起确定性转变 |
| TEMPO | 短暂出现（每次<1小时），有不确定性 |
| BECMG | 逐步渐变 |

**信任度判断：** TEMPO 超过预报周期一半时间→实际可能是主导天气；相邻机场 TAF 不一致→至少一个不准。

**放行策略：** 看最新 TAF + 上一份 TAF（6小时前），对比变化趋势。

---

### 3.5 SIGMET 与航路气象风险

**SIGMET 覆盖：** 雷暴（TS）、严重颠簸（SEV TURB）、严重积冰（SEV ICE）、火山灰云。
**AIRMET 覆盖：** 中低强度天气（低空/通航）。

| 颠簸等级 | 座舱感知 |
|----------|----------|
| 轻度（LGT） | 杯水面波动 |
| 中度（MOD） | 杯水溅出，行走困难 |
| 严重（SEV） | 人员被甩向安全带 |
| 极度（EXT） | 结构受力，短暂失控 |

**结冰条件：** 温度-露点≤3°C 且飞机将进入云层 = 结冰风险高。

---

### 3.6 雷暴绕飞

**雷达限制：** 气象雷达探测降水粒子回波，看不到 CAT、无冰晶云、远距离上冲气流有限。

**绕飞策略：**
| 策略 | 风险 |
|------|------|
| 上风方向绕飞 | 更少颠簸，燃油消耗多 |
| 下风方向绕飞 | 遇到更强烈颠簸 |
| 两雷暴间穿过 | 间隔≥40nm 为安全 |
| 从顶部飞越 | 需飞机升限允许 |
| 等待过境 | 时间成本 |

**核心规则：** 红色回波区域绕飞至少 20nm。

---

### 3.7 起降阶段气象决策清单

进近简令中逐项确认：当前 METAR → 前一份对比 → 目的地 TAF → 备降场 TAF → ATIS → 前机 PIREP。

**刹车效应：** GOOD / GOOD-MEDIUM / MEDIUM / MEDIUM-POOR / POOR。多机组报告 POOR 时使用 MAX 刹车或选择更长跑道。

## 四、空中交通管制

### 4.1 READ BACK（复诵）与 HEAR BACK（监听确认）

#### 4.1.1 定义

**READ BACK（复诵）**：飞行员收到管制指令后，逐字重复指令要素的过程。本质是一个认知校验——用发声输出验证输入信息的正确性。

**HEAR BACK（监听确认）**：管制员收到复诵后核实其正确性的责任。复诵有误时必须纠正（"Negative, I say again..."）。

#### 4.1.2 必须复诵的指令类型

- 跑道相关指令（进入、穿越、等待、起飞、着陆）
- 高度、航向、速度指令
- 应答机编码
- QNH / QFE
- 频率更换

"Roger" 或 "Wilco" 对此类指令无效。

#### 4.1.3 抄收与复诵对比

| 维度 | 抄收 | 复诵 |
|------|------|------|
| 回答用语 | Roger / Wilco | 逐字重复指令要素 |
| 解决的问题 | "我听到了吗" | "我听对了吗" |
| 是否包含参数 | 否 | 必须包含所有数字 |
| 适用场景 | 非关键信息 | 上述五类关键指令 |

#### 4.1.4 READ BACK 常见陷阱

| 陷阱 | 表现 | 后果 |
|------|------|------|
| Roger 替代复诵 | 用 Roger 回应关键指令 | 管制员无法确认参数正确性 |
| 部分复诵 | 漏掉某个要素 | 参数偏差影响安全 |
| 期望偏误 | 自动"补全"听到内容 | 复诵与指令不一致 |

### 4.2 CNS/ATM 架构

CNS/ATM（通信 Communication、导航 Navigation、监视 Surveillance / 空中交通管理 Air Traffic Management）是现代空域管理的技术基础。

| 支柱 | 技术 | 解决的问题 |
|------|------|-----------|
| 通信（C） | CPDLC | 文本化管制指令，消除语音歧义 |
| 导航（N） | PBN（RNAV/RNP） | 高精度航路保持 |
| 监视（S） | ADS-B / SSR | 高精度高更新率位置监视 |
| ATM | RVSM + 间隔标准 | 在保证安全前提下提高空域容量 |

### 4.3 洋区运行（CPDLC + ADS-C 工作流）

进入洋区前：CPDLC LOG ON → ADS-C 合同激活
巡航中：CPDLC 处理高度变更/通信，ADS-C 自动位置报告
退出洋区：CPDLC LOG OFF，转回 VHF 通信

## 五、航空法规与标准

### 5.1 ICAO 标准用语规范

#### 5.1.1 "cleared for takeoff" 的固定搭配

takeoff 只与 cleared 搭配使用，避免与 line up 混淆（ICAO Doc 9432）。

#### 5.1.2 频率读法规则

整数部分可分段读（118→"one one eight"或"one eighteen"）；小数部分用 decimal 逐位读（118.35→"decimal three five"）。

#### 5.1.3 "after departure" 的标准术语

离地后/离场后的管制术语用 after departure，非 after takeoff，非 after airborne。

### 5.2 ETOPS / EDTO 认证体系

EDTO（Extended Diversion Time Operations）的核心：确保飞机在航路上任意一点，单发失效后能在规定时间内飞到合适的备降场。

**三要素认证：**
1. **飞机**：关键系统冗余设计，ETOPS Significant System 清单
2. **发动机**：IFSD 率低于 0.02/1000 小时（ETOPS-180 标准）
3. **航空公司**：维修方案、签派能力、机组训练、MEL 管控

**备降场要求：** 合适的备降场（跑道长度/道面/导航设施）+ 可用的备降场（天气满足最低标准+进近程序+应急服务）。

## 六、飞行程序与操作

### 6.1 全飞行阶段系统概览

| 飞行阶段 | 涉及系统 |
|----------|----------|
| 推出/启动 | CPDLC（DCL数字放行） |
| 滑行 | GNSS 可靠性、低能见度运行 |
| 起飞 | PBN（RNP AR）、风切变、EGPWS Mode 3 |
| 离场爬升 | PBN（SID）、TCAS（TA/RA 高发期） |
| 航路巡航 | RVSM、ADS-B、CPDLC、ETOPS、TCAS、气象 |
| 下降/进近 | PBN（STAR）、EGPWS（全模式）、TCAS、气象 |
| 最后进近 | PBN（RNP APCH/AR）、EGPWS（Mode 5/6/7） |
| 着陆滑入 | EGPWS（SmartRunway/SmartLanding） |

### 6.2 ETOPS 延程运行

**ETOPS 等级以单发失效后飞至最近备降场的分钟数表示：**
| 等级 | 典型应用 |
|------|----------|
| ETOPS-120 | A320/737 跨水 |
| ETOPS-180 | 跨太平洋标准 |
| ETOPS-240/330 | A330/B777/B787 |

**四层防护体系：**
| 层级 | 系统 | 预警量级 |
|------|------|----------|
| 第一层 | 管制程序+CPDLC | 数分钟 |
| 第二层 | RVSM+ADS-B | 分钟级 |
| 第三层 | TCAS RA | 15-35 秒 |
| 第四层 | EGPWS PULL UP | 10-60 秒 |

### 6.3 关键系统交互场景

**场景：TCAS RA + GPWS PULL UP 同时触发**
- 方向一致 → 按指令执行
- RA"DESCEND" + PULL UP → 以 PULL UP 为准（爬升），地形冲突近在咫尺
